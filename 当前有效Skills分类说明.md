# 当前有效 Skills 分类说明

更新时间：2026-04-20  
当前有效 Skill 数量：44  
当前重复数量：0

这份文档按“更方便日常筛选和保留”的思路整理。  
分类以你给的三个方向为主：自定义内容、专有开发框架、前端 UI 相关；另外补了一个“文档 / 办公 / 多媒体”类，专门容纳表格、PPT、Obsidian、图片和视频相关技能。

## 1. 自定义内容

- `个人开发工作流`：这是你自己的工程工作规范包，核心作用不是生成某个文件，而是约束代理怎么调查、怎么找根因、怎么验证、怎么汇报。实际效果是让整个协作过程更贴合你的习惯，减少“看起来改了、其实没闭环”的情况。
- `langlang007-ghcr-release`：这是你自己的 GHCR 发布技能，专门服务 `langlang007` 的 Docker 镜像发版。它会围绕版本号、GitHub Actions、镜像推送和生产更新命令来组织流程，适合固定发布链路反复复用。
- `ssh-mcp-configurator`：这是你自己的 SSH MCP 配置技能，用来新增、重配、替换 Codex 可连接的远程服务器。它的实际价值是把实例名、认证方式、连接配置这些容易出错的步骤流程化。

## 2. 专有开发框架与工程工具

- `agent-reach`：这是联网搜索总路由，用来接网页、社媒、GitHub、视频、招聘等外部信息来源。它的原理是把不同场景路由到不同工具链，实际效果是查资料时更快找到合适入口，不用临时拼命令。
- `gitnexus-cli`：这是 GitNexus 的命令行操作技能，负责建索引、查状态、清索引、生成 wiki 这类基础动作。适合在正式分析前先把图谱环境准备好。
- `gitnexus-debugging`：这是用 GitNexus 查 bug 的技能，重点是沿调用链追“错误从哪里来、在哪一层开始歪掉”。面对跨模块问题时，它比单纯全文搜索更适合缩范围。
- `gitnexus-exploring`：这是理解代码结构和执行流的技能，适合回答“这块功能怎么跑起来”“谁在调用这个函数”。实际效果是更快建立系统级上下文，而不是只盯单个文件。
- `gitnexus-guide`：这是 GitNexus 的说明书技能，专门解释它有哪些工具、图谱 schema 怎么读、推荐工作流是什么。更像“教你怎么用 GitNexus 本身”。
- `gitnexus-impact-analysis`：这是改动风险分析技能，用来判断“如果动这个函数/模块，会牵动哪些上下游”。适合改共享代码、接口、通用组件前先看影响面。
- `gitnexus-refactoring`：这是安全重构技能，偏向改名、抽模块、拆函数、迁移结构这类改动。实际作用是让重构前先有依赖关系依据，减少误伤。
- `openai-docs`：这是 OpenAI 官方文档技能，主要用来查模型选择、API 用法、升级路径和最新官方建议。它的价值在于优先依赖官方口径，而不是旧记忆。
- `plugin-creator`：这是 Codex 插件骨架生成技能，适合新建本地插件目录、插件元数据和基本结构。它的实际效果是降低做插件的起步成本。
- `skill-creator`：这是“创建新 skill”用的技能，用来设计 skill 的结构、触发方式和说明文档。适合把一套常用经验沉淀成可复用能力。
- `skill-installer`：这是安装 skill 的技能，负责把外部 skill 从 curated list 或 GitHub 路径装进当前环境。适合扩容能力，而不是做业务功能本身。

## 3. 前端 UI 相关内容

- `adapt`：这是适配技能，用来把设计或页面调整到不同屏幕、设备、场景下都能成立。它的实际效果是降低“桌面好看、移动端崩掉”的风险。
- `animate`：这是动画和微交互技能，重点是让动效服务于反馈、节奏和可用性，而不是单纯加花活。适合给按钮、卡片、页面切换增加有目的的动效。
- `audit`：这是界面体检技能，会从可访问性、性能、响应式、主题一致性等角度检查页面。它产出的通常不是代码，而是一份偏审计视角的问题清单。
- `bolder`：这是“加冲击力”的技能，用来把太保守、太平的设计做得更有存在感。它更像视觉放大器，适合界面缺个性、缺记忆点的时候。
- `clarify`：这是文案澄清技能，专门改说明文字、标签、错误提示和操作引导。实际效果是把“用户看不懂”的地方翻成更直白、更不容易误解的表达。
- `colorize`：这是配色增强技能，用来给过于灰、过于单调的界面加层次和重点。它的效果通常是让界面更有视觉区分度，而不是简单“变花”。
- `critique`：这是设计评审技能，偏 UX 视角去看层级、信息结构、交互是否合理。适合做“这个界面到底哪里别扭”的诊断。
- `delight`：这是增加趣味和记忆点的技能，会引入一点小惊喜、小个性或更有温度的体验。它的作用不是解决功能问题，而是提升用户感受。
- `distill`：这是做减法的技能，适合把复杂界面收干净，把不必要的层级和视觉噪音拿掉。它的实际效果通常是“更清楚、更利落”。
- `extract`：这是设计系统抽取技能，用来从现有页面中沉淀可复用组件、token 和模式。适合从零散页面往系统化设计过渡。
- `frontend-design`：这是主力前端界面生成技能，偏向直接把页面和组件做出来，而且要有较好的视觉完成度。适合“做一个页面 / 做一个界面 / 做一个前端作品”这类任务。
- `harden`：这是稳健性增强技能，关注错误状态、边界情况、超长文本、i18n 等真实环境问题。实际效果是界面在脏数据或异常输入下更不容易坏。
- `normalize`：这是统一化技能，用来把页面拉回设计系统和统一风格上。适合设计已经有点散、同类组件长得不太像的时候。
- `onboard`：这是 onboarding 技能，专门优化首次使用体验、空状态、引导路径和新手理解成本。适合产品“用户一进来不知道干嘛”的场景。
- `optimize`：这是前端性能优化技能，通常关注加载、渲染、动画开销、图片体积和包大小。它的目标是让界面更轻、更快，而不是更花。
- `polish`：这是收尾打磨技能，偏最后一轮的细节对齐、间距调整、边角处理和一致性修正。适合页面已经差不多完成，但还差一点“成品感”的时候。
- `quieter`：这是降噪技能，用来把过于强势、过于炸的设计压下来。适合视觉元素互相抢戏、页面让人觉得累的时候。
- `shoelace-ui`：这是 Shoelace UI 专项技能，适合项目本来就用 Shoelace 组件和静态 HTML/CSS/JS 这套栈。它的价值在于顺着现有技术栈做，不会强行套别的组件体系。
- `teach-impeccable`：这是一次性设计上下文采集技能，会把项目的审美偏好和设计约束写进 AI 配置。它的实际效果是让后续页面产出更稳定，不会每次都换风格。
- `UI 设计大师`：这是偏“设计专家”和“架构顾问”定位的技能，比普通界面生成更强调布局策略、设计系统、双主题和工程可落地性。适合先定方向、再做实现的场景。
- `vibe-tech-pm`：这是把模糊需求翻译成高密度提示词的技能，擅长处理“高级一点”“酷一点”“好看一点”这类模糊描述。它会把这些感性要求拆成布局、材质、动效、排版等更可执行的方向。
- `web-design-guidelines`：这是按 Web 设计规范做 review 的技能，偏向检查界面有没有违反常见交互和可访问性原则。它更像一个 UI 规范审查员，而不是直接写页面的技能。

## 4. 文档 / 办公 / 多媒体 / 知识管理

- `Excel`：这是表格处理技能，适合新建、修改、分析和美化 Excel、CSV、TSV 这类数据文件。实际效果是能把“表格工作”从纯手工改成结构化处理。
- `imagegen`：这是图片生成与图片编辑技能，用来做插图、位图素材、贴图、透明背景 cutout 等视觉资产。适合需要“出图”，而不是“出页面代码”的任务。
- `json-canvas`：这是 Obsidian `.canvas` 技能，处理节点、连线、分组和画布布局。适合做知识图谱、脑图、流程图类画布结构。
- `mermaid-ascii-render`：这是 Mermaid 文本图转终端字符图的技能，适合在没有图形环境时快速看流程图。实际效果是把图形内容压缩成终端可读版本。
- `obsidian-bases`：这是 Obsidian `.base` 视图技能，偏“笔记数据库”的筛选、汇总、表格视图和公式配置。适合知识库结构化管理。
- `obsidian-markdown`：这是 Obsidian 风格 Markdown 技能，专门处理 wikilink、callout、frontmatter、embed 等语法。适合维护 Obsidian 笔记仓，而不是普通 Markdown 文档。
- `PowerPoint`：这是 PPT 生产技能，用来创建、编辑、渲染检查并导出可编辑 `.pptx`。适合正式汇报材料，而不是只产静态图片。
- `remotion-best-practices`：这是 Remotion 视频制作技能，适合用 React 程序化做字幕视频、动画片头、模板视频等内容。它会覆盖时间轴、字幕、音频、转场、素材导入等视频工作流问题。
